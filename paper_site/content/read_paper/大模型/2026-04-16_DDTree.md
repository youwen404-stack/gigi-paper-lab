# Accelerating Speculative Decoding with Block Diffusion Draft Trees

- Category: 大模型
- Read date: 2026-04-16
- Paper date: 2026-04-15
- Authors: Liran Ringel, Yaniv Romano
- Venue/Status: arXiv preprint
- Source: https://arxiv.org/abs/2604.12989
- Project: https://liranringel.github.io/ddtree/

## Why this paper

大模型方向这轮最值得补的一块，不是再看一个新的通用 benchmark，而是看 inference efficiency 正在怎么变化。DDTree 对 speculative decoding 做的改写比较干脆：不再让 drafter 逐 token 自回归地产生候选，而是用一次 block diffusion 直接生成一个 draft tree。

## Core problem

speculative decoding 已经是当前提升大模型生成速度的重要路线，但传统做法里 drafter 往往还是 autoregressive 的：

1. drafter 自己也要逐步生成多个 token。
2. 当温度升高或分支变多时，draft 质量和验证效率很难一起保持。
3. 草稿模型本身的串行性会吞掉一部分原本想节省的延迟。

所以问题不是“要不要 speculative decoding”，而是“能不能让 draft 生成本身也更并行”。

## Main idea

DDTree 的核心想法是用 block diffusion drafter 一次性预测下一段 token block 的位置分布，再把这些位置分布组织成一个 draft tree，然后交给 target model 一次前向验证。

可以把它理解成两步：

1. Block diffusion draft
drafter 不按 token 链式生成，而是一次性给出整个 block 中每个位置的候选分布。

2. Draft tree verification
系统从这组分布里构建树状候选，再让 target model 在一次 forward pass 里并行验证整棵树，而不是只验证单条草稿路径。

这让 speculative decoding 从“draft 多 token”进一步走向“draft 多路径”。

## What the paper claims

根据论文项目页与摘要，DDTree 在多个数据集上相对 autoregressive decoding 都给出明显加速，并且比强基线 DFlash 还更快。项目页展示了在 GSM8K、MATH-500 等任务上可达到 6x 到 8x 级别的速度提升，说明 block-level 并行 draft 在推理阶段确实有现实价值。

## Why it matters

这篇论文重要的地方在于，它把 speculative decoding 的瓶颈从“怎么让 verifier 更快”往前推了一步，直接重写了 drafter 的生成范式。

这背后有两个更值得关注的信号：

1. 推理优化开始越来越像“生成结构设计”问题，而不只是 kernel 或 cache 工程。
2. 草稿模型不必和 target model 使用同一种时间展开方式，异构生成机制可能会越来越重要。

## Quick takeaway

一句话总结：DDTree 试图把 speculative decoding 从“多 token 草稿”升级成“多路径、块级并行草稿”，从而进一步释放大模型推理加速空间。
