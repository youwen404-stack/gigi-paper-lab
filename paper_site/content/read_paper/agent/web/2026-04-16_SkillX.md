# SkillX: Automatically Constructing Skill Knowledge Bases for Agents

- Category: agent
- Read date: 2026-04-16
- Paper date: 2026-04-06
- Authors: Chenxi Wang, Zhuoyun Yu, Xin Xie, Wuguannan Yao, Runnan Fang, Shuofei Qiao, Kexin Cao, Guozhou Zheng, Xiang Qi, Peng Zhang, Shumin Deng
- Venue/Status: arXiv preprint
- Source: https://arxiv.org/abs/2604.04804
- PDF: https://arxiv.org/pdf/2604.04804

## Why this paper

这篇 paper 值得收，是因为它抓住了 agent 研究里一个很实用但经常被弱化的问题：经验能不能沉淀成可复用资产，而不是每个 agent 反复从头探索。相比单次任务成功率，这更接近 agent 系统能否持续进化的问题。

## Core problem

作者指出，现有 self-evolving LLM agents 往往存在三个低效点：

1. agent 彼此孤立学习，经验无法跨环境、跨任务复用。
2. 原始轨迹噪声大，难以直接转成稳定、可调用的技能表示。
3. 有限经验下容易重复探索，导致泛化和效率都不理想。

## Main method

SkillX 提出一个全自动技能库构建流程，目标是从 agent 轨迹里抽取可插拔的 SkillKB，再交给其他较弱 agent 复用。

方法包括三个核心模块：

1. Multi-Level Skills Design
把轨迹蒸馏成三级技能结构：strategic plans、functional skills、atomic skills，让经验表示从高层规划到低层动作都可复用。

2. Iterative Skills Refinement
根据执行反馈自动修订技能内容，持续提升技能描述质量和可执行性。

3. Exploratory Skills Expansion
主动生成并验证新技能，扩展初始数据覆盖范围，而不是只被动压缩已有轨迹。

## Reported results

根据 arXiv 摘要，作者使用强 backbone agent `GLM-4.6` 自动构建 SkillKB，并在 AppWorld、BFCL-v3、`\tau^2`-Bench 等长时程、用户交互 benchmark 上测试可迁移性。结果显示，把这个技能库接到较弱 base agents 上后，任务成功率与执行效率都能稳定提升。

## Quick takeaway

一句话总结：SkillX 试图把 agent 的“经验学习”从一次性轨迹模仿，推进成“自动抽取、迭代修正、跨 agent 复用”的技能资产构建流程。
